# ✅你平时用过哪些AI Coding的工具吗？



现在面试基本都会问这种问题了，因为用AI辅助编程已经是一个基本能力了，后面肯定会大规模的使用AI Coding，所以，面试会通过这个问题考察大家是否在积极拥抱AI。但是这个问题其实也不仅仅是为了考察你“用过多少种工具”，而是想考察以下三点：

1. **技术敏感度**：你是否关注前沿技术，是否愿意拥抱新工具提升效率。
2. **工程化思维**：你是把AI当作“抄代码的捷径”，还是当作“辅助思考、审查代码、编写测试”的结对编程伙伴
3. **边界意识**：你是否清楚AI的局限性（如幻觉、安全漏洞），是否有审查和验证AI生成代码的能力。



我见过很多人这么回答：



+ <font style="color:rgb(6, 10, 38);">“我大部分代码都是AI写的，我只负责复制粘贴。”</font>
    - **<font style="color:rgb(6, 10, 38);"></font>**<font style="color:rgb(6, 10, 38);">面试官会认为你缺乏独立 coding 能力，一旦AI出错你就束手无策。</font>
+ <font style="color:rgb(6, 10, 38);">“我没用过，我觉得没必要/那是作弊。”</font>
    - **<font style="color:rgb(6, 10, 38);"></font>**<font style="color:rgb(6, 10, 38);">显得你固步自封，不愿意接受新工具，团队协作效率可能较低。</font>
+ <font style="color:rgb(6, 10, 38);">“我用过 Github Copilot。”然后就没下文了。</font>
    - <font style="color:rgb(6, 10, 38);">浪费了一个展示你对工具的理解&使用以及优化思维的好机会。</font>

<font style="color:rgb(6, 10, 38);"></font>

# <font style="color:rgb(6, 10, 38);">典型回答</font>


目前在国内，用的比较多的AI Coding是下面这几个：



Claude Code

Open Code （开源版Claude Code）

Cursor

Github Copilot

Qoder

TRAE



这几个除了Open Code以外，基本都是需要付费的。



推荐的回答方式：



<font style="color:rgba(6, 10, 38, 0.7) !important;">“我主要用 </font>**<font style="color:rgb(6, 10, 38);">Cursor</font>**<font style="color:rgba(6, 10, 38, 0.7) !important;"> 进行日常的辅助编程和代码生成，特别是在写单元测试和重构时效率提升很明显，或者有的是涉及到一些比较简单的数据库层面的CRUD我也会让Cursor帮我生成一部分。但我始终坚持'Human-in-the-loop'，所有AI生成的代码我都会经过严格的Review和测试才合并，确保安全和逻辑正确。”</font>
